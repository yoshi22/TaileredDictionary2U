# 00. Overview - TaileredDictionary2U

## プロジェクト名
**TaileredDictionary2U** (TD2U)
- "Tailored" + "Dictionary" + "to You" の造語
- 個人に最適化された辞書・単語帳アプリ

---

## 価値仮説 (Value Hypothesis)

### 課題 (Problem)
1. **業界キャッチアップ**: 新しい業界・技術分野に入ると専門用語が多く、キャッチアップに時間がかかる
2. **オンボーディング**: 新しい環境（会社・プロジェクト）では固有の略語・用語があり、会議についていけない
3. **語学学習**: 単語帳を作っても復習が続かない、効率的な学習方法がわからない

### 解決策 (Solution)
- **Entry（用語）を登録**し、詳細画面から必要なタイミングでAIが「日英訳 + 3行要約 + 使用例 + 関連語 + 参考リンク候補」を生成
- **SRS（間隔反復）**で最適なタイミングで復習をリマインド
- **Webアプリを中心**に提供し、モバイル展開は段階的に進める

### 独自価値 (Unique Value Proposition)
> 「知らない言葉に出会ったら、登録するだけ。必要なときにAIとSRSにおまかせ。」

- 登録の手間を最小化（用語入力のみ）
- AIが文脈を理解して適切な情報を生成
- 科学的に実証されたSRSで記憶定着を最大化

---

## 非ゴール (Non-Goals)

### MVP段階では対応しない
| 項目 | 理由 |
|-----|------|
| 多言語対応（日英以外） | まず日英で価値検証 |
| 共有・コラボ機能 | 個人向けに集中 |
| オフライン完全対応 | 技術複雑度が高い |
| 音声読み上げ | 優先度低 |
| 画像・動画添付 | MVPスコープ外 |
| OCR・Share Sheet | モバイルフェーズで対応 |
| ソーシャル機能 | 個人向けに集中 |
| エンタープライズ機能 | B2C優先 |

### 将来的にも対応しない可能性
- 完全なオフラインAI生成（コスト・品質のバランス）
- リアルタイムコラボレーション（Notion/Google Docs的な）

## 用語定義 (Glossary)

| 用語 | 定義 |
|-----|------|
| **Entry** | ユーザーが登録する1つの学習単位（用語・フレーズ・文） |
| **Term** | Entry内の主要な単語・フレーズ（登録対象） |
| **Context** | Termの周辺文脈（前後1-2文、任意入力） |
| **Enrichment** | AIが生成した付加情報（翻訳・要約・例文など） |
| **SRS** | Spaced Repetition System（間隔反復学習システム） |
| **Review** | SRSに基づく復習セッション |
| **Ease Factor** | SRSの難易度係数（高いほど間隔が長くなる） |
| **Interval** | 次回復習までの日数 |
| **Due** | 復習予定日時 |
| **Deck** | Entryをグループ化するフォルダ的概念 |
| **Entitlement** | ユーザーが持つ権利・機能アクセス権 |
| **Credit** | AI生成に使用する消費型ポイント |
| **Generation** | AI呼び出しによるEnrichment生成 |

## 成功指標 (Success Metrics)

### North Star Metric
**週間アクティブ復習ユーザー数 (WAU-Reviewers)**
- 理由: 復習を続けているユーザーが価値を感じている証拠

### 補助指標
| 指標 | 目標（MVP 3ヶ月後） |
|-----|-----|
| D1 Retention | > 40% |
| D7 Retention | > 25% |
| D30 Retention | > 15% |
| Free → Plus CVR | > 3% |
| 平均Entry数/ユーザー | > 50 |
| 週間復習回数/ユーザー | > 3 |

## ステークホルダー

### MVP段階
- **プロダクトオーナー/開発者**: 1人（個人開発）
- **初期ユーザー**: 100-1000人（β期間）

### 将来
- **サポート**: ユーザーコミュニティ、FAQ
- **コンテンツ**: テンプレートDeck提供者（コミュニティ）

## 技術的仮定

| 仮定 | 根拠 |
|-----|------|
| GPT-4o-miniで十分な品質 | コスト効率 $0.001-0.003/Entry |
| SM-2ベースSRSで効果的 | Anki等で実証済み |
| Supabase Free tierで開始可能 | 初期1000ユーザーまで対応 |
| Next.js App Routerで十分 | Server Components活用 |
| Expoでモバイル展開可能 | Web版と同じSupabase利用 |
